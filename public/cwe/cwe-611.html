Why Is This an Issue?

CWE-611 covers XML External Entity (XXE) Injection — a vulnerability that occurs when an XML parser processes untrusted XML input containing references to external entities. Attackers can leverage this to read local files, probe internal networks (SSRF), or cause a Denial of Service.

XXE is particularly dangerous in Java because many legacy XML parsers (like DocumentBuilderFactory and SAXParserFactory) have external entity resolution enabled by default.

  • File Disclosure — Reading configuration files or secrets using the file:// protocol.
  • SSRF — Making the server perform requests to internal metadata or admin services.
  • Billion Laughs Attack — Causing an exponential memory exhaustion DOS via recursive entity expansion.

This weakness is part of OWASP A05:2021 — Security Misconfiguration. The primary defense is to disable DTD (Document Type Definition) and external entity processing in your XML parser configuration.

Examples of the Issue

Dangerous: XML Parsing with Default Configuration (Java / DocumentBuilder)

By default, DocumentBuilderFactory resolves external entities. An attacker can submit an XML payload that defines a custom entity pointing to a local file (like /etc/passwd) and then reference it in the XML body.

// DANGEROUS: Default configuration allows XXE. public void parseXml(InputStream is) throws Exception { DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance(); // No security features set — DTD processing is enabled by default! DocumentBuilder builder = factory.newDocumentBuilder(); Document doc = builder.parse(is); // If the XML is: // <?xml version="1.0" encoding="ISO-8859-1"?> // <!DOCTYPE foo [ <!ENTITY xxe SYSTEM "file:///etc/passwd" > ]> // <root>&xxe;</root> // The parser will read /etc/passwd and include it in 'doc' }

Dangerous: XML Parsing with Entity Resolution (Node.js / libxmljs)

JavaScript parsers like libxmljs have options to enable entity resolution. If noent: true is passed, the parser becomes vulnerable to XXE.

// DANGEROUS: 'noent: true' enables external entity resolution. const libxmljs = require("libxmljs"); app.post('/upload-xml', (req, res) => { const xml = req.body.data; const doc = libxmljs.parseXml(xml, { noent: true, // ← This enables XXE vulnerability! dtdvalid: false }); res.send(doc.toString()); });

Dangerous: Unsecured SAX or StAX Parsers (Java)

Stream-based parsers like SAXParser and XMLInputFactory (StAX) are also vulnerable if features like DISALLOW_DOCTYPE_DECL are not manually set.

// DANGEROUS: Unsecured XMLInputFactory. XMLInputFactory factory = XMLInputFactory.newInstance(); // Default values allow DTD and external entities! XMLStreamReader reader = factory.createXMLStreamReader(new FileInputStream("data.xml"));

How to Resolve

Fix (Java): Fully Disabling DTD and External Entities

The most recommended defense is to completely disable DTDs. If the parser cannot process DTDs, it cannot process external entities. This is the "Nuclear Option" and protects against both XXE and XML Denial of Service.

// SAFE: Securely configuring DocumentBuilderFactory. DocumentBuilderFactory dbf = DocumentBuilderFactory.newInstance(); try { // 1. Disable DTDs entirely (Best protection) String FEATURE = "http://apache.org/xml/features/disallow-doctype-decl"; dbf.setFeature(FEATURE, true); // 2. If you MUST support DTDs, disable external entity resolution specifically: dbf.setFeature("http://xml.org/sax/features/external-general-entities", false); dbf.setFeature("http://xml.org/sax/features/external-parameter-entities", false); dbf.setFeature("http://apache.org/xml/features/nonvalidating/load-external-dtd", false); // 3. Optional: Disable Billion Laughs (entity expansion limit) dbf.setXIncludeAware(false); dbf.setExpandEntityReferences(false); } catch (ParserConfigurationException e) { // This can happen if the parser doesn't support the feature } DocumentBuilder builder = dbf.newDocumentBuilder();

Fix (JS): Secure XML parsing options

In Node.js, ensure that noent is set to false (or omitted, as it's the default in most modern versions) and that external entity resolution is disabled.

// SAFE: Default parsing (noent is false by default). const libxmljs = require("libxmljs"); // Do NOT set noent: true. const doc = libxmljs.parseXml(xml); // Safe by default. // If you really need DTD validation, use a safe validator that doesn't // resolve network-based entities.

Defense-in-Depth: Prefer JSON or newer XML APIs

The best way to prevent XXE is to avoid XML entirely in favor of simpler formats like JSON or Protocol Buffers. If XML is required, use modern APIs that are secure by default, such as Jackson XML (which doesn't process DTDs by default).

// SAFE: Using Jackson XML (Secure by default). XmlMapper xmlMapper = new XmlMapper(); // Jackson does not enable DTD processing or external entities by default. MyData data = xmlMapper.readValue(is, MyData.class);
