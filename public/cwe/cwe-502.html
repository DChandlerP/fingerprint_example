Why Is This an Issue?

CWE-502 flags code that deserializes objects from untrusted sources without verification. When an application calls ObjectInputStream.readObject() on attacker-controlled data, the JVM instantiates whatever class the byte stream specifies — including classes with dangerous side effects in their readObject(), readResolve(), or finalize() methods.

This is not a theoretical risk. Public exploit chains (called "gadget chains") exist for virtually every major Java framework — Apache Commons Collections, Spring, Hibernate, and many more. Tools like ysoserial can generate payloads that achieve Remote Code Execution (RCE) with a single crafted byte stream. The 2017 Equifax breach was enabled in part by a Java deserialization vulnerability.

This weakness is ranked in the SANS/CWE Top 25 and maps to OWASP A08:2021 — Software and Data Integrity Failures. The safest approach is to never deserialize untrusted data with native Java serialization. Use safe data formats like JSON instead.

Examples of the Issue

Dangerous: Deserializing an object directly from an HTTP request body

This servlet reads raw bytes from the HTTP request and feeds them directly to ObjectInputStream.readObject(). An attacker sends a crafted serialized payload (e.g., generated by ysoserial) that triggers a gadget chain on deserialization — achieving remote code execution before any application logic runs.

// DANGEROUS: Deserializing untrusted data from the network. @WebServlet("/import") public class ImportServlet extends HttpServlet { protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { // The attacker controls every byte of this stream. try (ObjectInputStream ois = new ObjectInputStream(req.getInputStream())) { Object obj = ois.readObject(); // RCE: gadget chain executes here UserProfile profile = (UserProfile) obj; profileService.save(profile); resp.getWriter().write("Import successful."); } catch (ClassNotFoundException e) { resp.sendError(400, "Invalid data."); } } }

Dangerous: Deserializing a Base64-encoded Java object from a cookie

Some applications store serialized Java objects in cookies for "stateless" session management. The cookie value is Base64-encoded but not signed or encrypted. An attacker simply replaces the cookie with a malicious serialized payload — the server deserializes it on the next request and the gadget chain fires.

// DANGEROUS: Deserializing a user-controlled cookie value. Cookie[] cookies = request.getCookies(); for (Cookie cookie : cookies) { if ("session_obj".equals(cookie.getName())) { byte[] data = Base64.getDecoder().decode(cookie.getValue()); try (ObjectInputStream ois = new ObjectInputStream( new ByteArrayInputStream(data))) { // RCE: attacker controls the cookie value. SessionData session = (SessionData) ois.readObject(); request.setAttribute("session", session); } } }

Dangerous: Deserializing uploaded files with ObjectInputStream

An application accepts .dat file uploads and deserializes them to restore saved state. The file extension check provides zero security — the attacker simply uploads a valid .dat file containing a ysoserial gadget chain payload.

// DANGEROUS: Deserializing uploaded files without any integrity check. @PostMapping("/upload-config") public ResponseEntity<String> uploadConfig(@RequestParam("file") MultipartFile file) throws IOException, ClassNotFoundException { String filename = file.getOriginalFilename(); if (filename == null || !filename.endsWith(".dat")) { return ResponseEntity.badRequest().body("Only .dat files accepted."); } // Extension check is meaningless — the bytes inside are what matter. try (ObjectInputStream ois = new ObjectInputStream(file.getInputStream())) { AppConfig config = (AppConfig) ois.readObject(); // RCE on deserialization configService.apply(config); } return ResponseEntity.ok("Configuration loaded."); }

How to Resolve

Fix: Replace Java serialization with JSON (Jackson / Gson)

The most effective fix is to stop using ObjectInputStream entirely. JSON libraries like Jackson or Gson deserialize data into explicit, known classes — they cannot instantiate arbitrary objects or trigger gadget chains. This eliminates the entire attack surface.

// SAFE: Jackson maps JSON properties to a specific, known class. // No arbitrary object instantiation, no gadget chains. import com.fasterxml.jackson.databind.ObjectMapper; @WebServlet("/import") public class ImportServlet extends HttpServlet { private final ObjectMapper mapper = new ObjectMapper(); protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { // Jackson only populates fields defined on UserProfile. // Unknown properties are rejected, not executed. UserProfile profile = mapper.readValue(req.getInputStream(), UserProfile.class); // Additional validation if (profile.getUsername() == null || profile.getUsername().isBlank()) { resp.sendError(400, "Username is required."); return; } profileService.save(profile); resp.getWriter().write("Import successful."); } }

Fix: If Java serialization is unavoidable, use ObjectInputFilter (JEP 290)

Java 9+ (and backports to 8u121+) provide ObjectInputFilter, which lets you allowlist the exact classes that may be deserialized. Any class not on the list is rejected before instantiation. This blocks gadget chains because the exploit classes (e.g., InvokerTransformer) are never allowed through the filter.

// SAFE: Only explicitly allowed classes can be deserialized. import java.io.ObjectInputFilter; ObjectInputStream ois = new ObjectInputStream(inputStream); // Allowlist: ONLY these classes may be deserialized. // Everything else — including gadget chain classes — is rejected. ObjectInputFilter filter = ObjectInputFilter.Config.createFilter( "com.myapp.model.UserProfile;" + "com.myapp.model.Address;" + "java.lang.String;" + "java.lang.Number;" + "java.util.ArrayList;" + "!*" // Reject everything else ); ois.setObjectInputFilter(filter); try { UserProfile profile = (UserProfile) ois.readObject(); // Safe: filtered profileService.save(profile); } catch (InvalidClassException e) { log.warn("Blocked deserialization of unauthorized class: {}", e.getMessage()); throw new SecurityException("Invalid data format."); }

Fix: Sign serialized data with HMAC before storing, verify before deserializing

If you must store serialized Java objects (e.g., in a database or message queue), sign them with an HMAC at write time and verify the signature before deserializing. If the signature doesn't match, the data has been tampered with and must be rejected. This prevents an attacker from substituting a malicious payload.

// SAFE: Verify HMAC signature BEFORE deserializing. import javax.crypto.Mac; import javax.crypto.spec.SecretKeySpec; public class SecureDeserializer { private static final String HMAC_ALGO = "HmacSHA256"; private final SecretKeySpec signingKey; public SecureDeserializer(byte[] secret) { this.signingKey = new SecretKeySpec(secret, HMAC_ALGO); } /** Serialize + sign: store both the data and the HMAC. */ public byte[] serializeAndSign(Serializable obj) throws Exception { ByteArrayOutputStream baos = new ByteArrayOutputStream(); try (ObjectOutputStream oos = new ObjectOutputStream(baos)) { oos.writeObject(obj); } byte[] data = baos.toByteArray(); byte[] hmac = computeHmac(data); // Prepend 32-byte HMAC to the serialized data return ByteBuffer.allocate(32 + data.length).put(hmac).put(data).array(); } /** Verify HMAC first, then deserialize only if authentic. */ public Object verifyAndDeserialize(byte[] signed) throws Exception { if (signed.length < 32) throw new SecurityException("Data too short."); byte[] storedHmac = Arrays.copyOfRange(signed, 0, 32); byte[] data = Arrays.copyOfRange(signed, 32, signed.length); byte[] computedHmac = computeHmac(data); if (!MessageDigest.isEqual(storedHmac, computedHmac)) { throw new SecurityException("HMAC verification failed — data tampered."); } try (ObjectInputStream ois = new ObjectInputStream( new ByteArrayInputStream(data))) { return ois.readObject(); // Safe: signature verified } } private byte[] computeHmac(byte[] data) throws Exception { Mac mac = Mac.getInstance(HMAC_ALGO); mac.init(signingKey); return mac.doFinal(data); } }
