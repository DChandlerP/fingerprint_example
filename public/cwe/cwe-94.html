Why Is This an Issue?

CWE-94 flags code that constructs and executes program logic from untrusted input at runtime. Unlike command injection (CWE-78), which targets the operating system, code injection targets the application's own runtime — allowing an attacker to execute arbitrary Java, JavaScript, or other interpreted code within the application's process.

Common attack surface includes calls to eval(), ScriptEngine.eval(), new Function(), or any mechanism that interprets a string as executable code. When the string is influenced by user input, an attacker can inject logic to steal data, bypass authentication, modify application state, or pivot to further attacks.

This weakness is ranked in the SANS/CWE Top 25 and maps to OWASP A03:2021 — Injection. The fix is straightforward: never execute untrusted strings as code. Use safe parsers, allowlists, or purpose-built libraries instead.

Examples of the Issue

Dangerous: Java ScriptEngine.eval() with user-supplied expression

A servlet accepts a "calculator" expression from the user and evaluates it with the built-in JavaScript engine. An attacker can submit a payload like java.lang.Runtime.getRuntime().exec("rm -rf /") instead of a math expression — gaining full remote code execution on the server.

// DANGEROUS: User input is executed as code by the ScriptEngine. @WebServlet("/calculate") public class CalcServlet extends HttpServlet { protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { String expression = req.getParameter("expr"); // e.g. "2+2" or malicious code! ScriptEngineManager mgr = new ScriptEngineManager(); ScriptEngine engine = mgr.getEngineByName("js"); try { Object result = engine.eval(expression); // RCE: attacker controls the code resp.getWriter().write("Result: " + result); } catch (ScriptException e) { resp.getWriter().write("Error: " + e.getMessage()); } } }

Dangerous: Node.js eval() with user input

A Node.js/Express route uses eval() to dynamically compute a value from a query parameter. An attacker can inject require('child_process').execSync('cat /etc/passwd') to execute arbitrary system commands on the server.

// DANGEROUS: eval() executes arbitrary code from the query string. const express = require('express'); const app = express(); app.get('/calc', (req, res) => { const expr = req.query.expr; // e.g. "2+2" or malicious code! try { const result = eval(expr); // RCE: attacker controls the code res.json({ result }); } catch (e) { res.status(400).json({ error: e.message }); } });

Dangerous: new Function() with user-controlled body

new Function() is functionally identical to eval() — it compiles a string into executable code. In this example, an API endpoint builds a discount-calculation function from user input, allowing an attacker to inject arbitrary logic that runs with the application's full privileges.

// DANGEROUS: new Function() compiles user input into executable code. app.post('/discount', (req, res) => { const formula = req.body.formula; // e.g. "return price * 0.9" or malicious code! // Attacker can submit: "return require('fs').readFileSync('/etc/passwd','utf8')" const calcDiscount = new Function('price', formula); const finalPrice = calcDiscount(100); res.json({ finalPrice }); });

How to Resolve

Fix (Java): Replace ScriptEngine with a safe expression parser

Instead of evaluating arbitrary code, use a purpose-built math expression library like exp4j that only understands numbers and arithmetic operators. It cannot execute Java classes, access the filesystem, or perform any action beyond basic math — completely eliminating the code injection vector.

// SAFE: exp4j only parses mathematical expressions — no code execution. import net.objecthunter.exp4j.Expression; import net.objecthunter.exp4j.ExpressionBuilder; @WebServlet("/calculate") public class CalcServlet extends HttpServlet { protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { String expression = req.getParameter("expr"); // Allowlist: reject anything that isn't a number, operator, or parenthesis. if (!expression.matches("[0-9+\\-*/().\\s]+")) { resp.sendError(400, "Invalid expression."); return; } try { Expression exp = new ExpressionBuilder(expression).build(); double result = exp.evaluate(); resp.getWriter().write("Result: " + result); } catch (Exception e) { resp.sendError(400, "Could not evaluate expression."); } } }

Fix (JS): Replace eval() with a safe math parser or JSON.parse()

For math expressions, use a dedicated library like mathjs that parses expressions without executing arbitrary code. For data, use JSON.parse() which only deserializes data and never executes code. Pair these with a strict Content-Security-Policy header that blocks unsafe-eval in the browser.

// SAFE: mathjs only evaluates mathematical expressions, not arbitrary code. const { evaluate } = require('mathjs'); app.get('/calc', (req, res) => { const expr = req.query.expr; // Allowlist: reject anything that isn't a number, operator, or parenthesis. if (!/^[0-9+\-*/().\s]+$/.test(expr)) { return res.status(400).json({ error: 'Invalid expression' }); } try { const result = evaluate(expr); // Safe: cannot execute system commands res.json({ result }); } catch (e) { res.status(400).json({ error: 'Could not evaluate expression' }); } }); // For data parsing, use JSON.parse() — it never executes code: // const data = JSON.parse(req.body.payload); // Safe

Fix (JS): Replace new Function() with a lookup map / strategy pattern

Instead of dynamically compiling code from user input, define all allowed behaviors ahead of time in a lookup map. The user selects a named strategy (e.g., "percent_off"), and the server executes the corresponding pre-written function. Unknown keys are rejected — no code is ever constructed from input.

// SAFE: Pre-defined discount strategies — no dynamic code generation. const discountStrategies = { 'percent_10': (price) => price * 0.90, 'percent_20': (price) => price * 0.80, 'flat_5': (price) => price - 5, 'bogo': (price) => price / 2, }; app.post('/discount', (req, res) => { const strategyName = req.body.strategy; // e.g. "percent_10" const calcDiscount = discountStrategies[strategyName]; if (!calcDiscount) { return res.status(400).json({ error: 'Unknown discount strategy' }); } const finalPrice = calcDiscount(100); res.json({ finalPrice }); });
