Why Is This an Issue?

CWE-327 flags code that uses cryptographic algorithms known to be broken, weak, or deprecated. These algorithms were once considered secure but have been defeated by advances in cryptanalysis and computing power. Common offenders include:

  • MD5 — collision attacks demonstrated in seconds; used to forge SSL certificates
  • SHA-1 — practical collision attacks since 2017 (SHAttered); deprecated by NIST, browsers, and CAs
  • DES / 3DES — 56-bit (DES) and 112-bit effective (3DES) key lengths; vulnerable to brute force and Sweet32 attacks
  • RC4 — statistical biases in output; prohibited by RFC 7465 in TLS
  • MD5 / SHA-1 for password hashing — far too fast; billions of hashes per second on a GPU

Using these algorithms gives a false sense of security. Data that appears encrypted or integrity-protected is actually vulnerable to forgery, collision attacks, or brute-force decryption. This is especially dangerous for password hashing, digital signatures, and data-at-rest encryption.

This weakness maps to OWASP A02:2021 — Cryptographic Failures. The fix is to replace broken algorithms with their modern, NIST-recommended counterparts: SHA-256 or SHA-3 for hashing, AES-256 for encryption, and bcrypt / Argon2 for password storage.

Examples of the Issue

Dangerous: MD5 used for password hashing (Java)

MD5 produces collisions in seconds and can be brute-forced at billions of hashes per second on consumer GPUs. Using it to hash passwords means an attacker with a stolen database can recover most passwords in minutes using rainbow tables or hashcat. Even adding a salt does not fix the core problem — MD5 is simply too fast for password storage.

// DANGEROUS: MD5 is broken — trivially reversible for passwords. import java.security.MessageDigest; public class AuthService { public String hashPassword(String password) throws Exception { // MD5: 128-bit output, known collision attacks, GPU-crackable. MessageDigest md = MessageDigest.getInstance("MD5"); byte[] hash = md.digest(password.getBytes(StandardCharsets.UTF_8)); return HexFormat.of().formatHex(hash); } public boolean verifyPassword(String password, String storedHash) throws Exception { return hashPassword(password).equals(storedHash); } } // Also dangerous — SHA-1 has the same speed problem for passwords: MessageDigest sha1 = MessageDigest.getInstance("SHA-1"); // Don't use for passwords!

Dangerous: SHA-1 for data integrity and HMAC signing (Node.js)

SHA-1 has been broken since the SHAttered attack in 2017, which produced two different PDF files with the same SHA-1 hash. Using SHA-1 for HMAC signatures or data integrity checks means an attacker could craft a malicious payload that produces the same hash as a legitimate one — bypassing signature verification.

// DANGEROUS: SHA-1 is broken — collision attacks are practical. const crypto = require('crypto'); /** Verify file integrity using SHA-1. */ function verifyChecksum(data, expectedHash) { // SHA-1: 160-bit output, practical collisions demonstrated. const hash = crypto.createHash('sha1').update(data).digest('hex'); return hash === expectedHash; } /** Sign an API request with HMAC-SHA1. */ function signRequest(payload, secret) { // HMAC-SHA1 is less affected than bare SHA-1, but still flagged // as risky and should be replaced with HMAC-SHA256. return crypto.createHmac('sha1', secret) .update(JSON.stringify(payload)) .digest('hex'); }

Dangerous: DES and DESede (3DES) for data encryption (Java)

DES uses a 56-bit key that can be brute-forced in hours on modern hardware. 3DES (DESede) has an effective key length of only 112 bits and a 64-bit block size — making it vulnerable to the Sweet32 birthday attack when encrypting large amounts of data. Both were officially retired by NIST in 2023.

// DANGEROUS: DES has a 56-bit key — brute-forceable in hours. public class LegacyEncryption { public static byte[] encryptWithDES(byte[] data, SecretKey key) throws Exception { Cipher cipher = Cipher.getInstance("DES/CBC/PKCS5Padding"); // 56-bit key! cipher.init(Cipher.ENCRYPT_MODE, key); return cipher.doFinal(data); } // 3DES is also deprecated — 64-bit block size enables Sweet32 attacks. public static byte[] encryptWith3DES(byte[] data, SecretKey key) throws Exception { Cipher cipher = Cipher.getInstance("DESede/CBC/PKCS5Padding"); cipher.init(Cipher.ENCRYPT_MODE, key); return cipher.doFinal(data); } // RC4 (Arcfour) is broken — statistical biases and prohibited in TLS. public static byte[] encryptWithRC4(byte[] data, SecretKey key) throws Exception { Cipher cipher = Cipher.getInstance("RC4"); // Prohibited by RFC 7465! cipher.init(Cipher.ENCRYPT_MODE, key); return cipher.doFinal(data); } }

How to Resolve

Fix (Java): Use bcrypt for password hashing

Replace MD5/SHA-1 with bcrypt for password storage. Bcrypt is intentionally slow — its cost factor makes brute-force attacks impractical even with GPUs. It automatically generates and embeds a random salt, and the cost factor can be increased over time as hardware improves.

// SAFE: bcrypt is purpose-built for password hashing. // Add dependency: org.mindrot:jbcrypt:0.4 import org.mindrot.jbcrypt.BCrypt; public class AuthService { // Cost factor 12 = ~250ms per hash on modern hardware. // Increase as hardware gets faster. private static final int BCRYPT_COST = 12; /** Hash a password for storage. */ public String hashPassword(String password) { // BCrypt.gensalt() generates a random 128-bit salt automatically. return BCrypt.hashpw(password, BCrypt.gensalt(BCRYPT_COST)); } /** Verify a password against its stored bcrypt hash. */ public boolean verifyPassword(String password, String storedHash) { return BCrypt.checkpw(password, storedHash); } } // Alternative: Argon2 (winner of the Password Hashing Competition) // Add dependency: de.mkammerer:argon2-jvm:2.11 // Argon2id is the recommended variant — resistant to both GPU and side-channel attacks.

Fix (JS): SHA-256 for integrity, HMAC-SHA256 for signing, bcrypt for passwords

Replace SHA-1 with SHA-256 (or SHA-3) for data integrity and HMAC signing. For password hashing, use bcrypt — never a raw hash function, regardless of how strong it is.

// SAFE: SHA-256 for integrity, bcrypt for passwords. const crypto = require('crypto'); const bcrypt = require('bcrypt'); /** Verify file integrity using SHA-256. */ function verifyChecksum(data, expectedHash) { const hash = crypto.createHash('sha256').update(data).digest('hex'); // Use timingSafeEqual to prevent timing attacks on hash comparison. return crypto.timingSafeEqual( Buffer.from(hash, 'hex'), Buffer.from(expectedHash, 'hex') ); } /** Sign an API request with HMAC-SHA256. */ function signRequest(payload, secret) { return crypto.createHmac('sha256', secret) .update(JSON.stringify(payload)) .digest('hex'); } /** Hash a password for storage using bcrypt. */ async function hashPassword(password) { const SALT_ROUNDS = 12; return bcrypt.hash(password, SALT_ROUNDS); } /** Verify a password against its stored bcrypt hash. */ async function verifyPassword(password, storedHash) { return bcrypt.compare(password, storedHash); }

Fix (Java): Replace DES/3DES/RC4 with AES-256-GCM

Replace all legacy ciphers with AES/GCM/NoPadding using a 256-bit key. AES-256 is approved by NIST for TOP SECRET classification and is the universal standard for symmetric encryption. GCM mode provides authenticated encryption — both confidentiality and integrity.

// SAFE: AES-256-GCM replaces all legacy ciphers (DES, 3DES, RC4). import javax.crypto.Cipher; import javax.crypto.KeyGenerator; import javax.crypto.SecretKey; import javax.crypto.spec.GCMParameterSpec; import java.security.SecureRandom; public class ModernEncryption { private static final String ALGORITHM = "AES/GCM/NoPadding"; private static final int GCM_IV_LENGTH = 12; private static final int GCM_TAG_BITS = 128; /** Generate a 256-bit AES key. */ public static SecretKey generateKey() throws Exception { KeyGenerator keyGen = KeyGenerator.getInstance("AES"); keyGen.init(256, new SecureRandom()); return keyGen.generateKey(); } public static byte[] encrypt(byte[] plaintext, SecretKey key) throws Exception { byte[] iv = new byte[GCM_IV_LENGTH]; new SecureRandom().nextBytes(iv); Cipher cipher = Cipher.getInstance(ALGORITHM); cipher.init(Cipher.ENCRYPT_MODE, key, new GCMParameterSpec(GCM_TAG_BITS, iv)); byte[] ciphertext = cipher.doFinal(plaintext); // Prepend IV for decryption. byte[] output = new byte[iv.length + ciphertext.length]; System.arraycopy(iv, 0, output, 0, iv.length); System.arraycopy(ciphertext, 0, output, iv.length, ciphertext.length); return output; } } // Quick reference — algorithm replacement guide: // MD5 → SHA-256 (or SHA-3-256) // SHA-1 → SHA-256 (or SHA-3-256) // DES → AES-256-GCM // 3DES/DESede → AES-256-GCM // RC4 → AES-256-GCM (or ChaCha20-Poly1305) // MD5/SHA-1 for passwords → bcrypt or Argon2id
