Why Is This an Issue?

CWE-326 flags the use of encryption configurations that do not provide adequate security. When SAST tools report "Inadequate Padding for AES encryption", they are identifying code that uses AES with a weak or dangerous mode of operation or padding scheme, such as:

  • ECB mode — encrypts each block independently, leaking patterns in the plaintext (the classic "ECB penguin" problem)
  • CBC without integrity — vulnerable to padding oracle attacks that can decrypt ciphertext without the key
  • NoPadding with block ciphers — requires exact block-size inputs, causing data loss or crashes
  • PKCS5Padding with CBC and no HMAC — enables bit-flipping and padding oracle attacks

Even though AES itself is a strong cipher, choosing the wrong mode or padding scheme can make the encryption trivially breakable. ECB mode reveals identical plaintext blocks as identical ciphertext blocks. CBC mode without authentication allows attackers to manipulate ciphertext and recover plaintext through padding oracle attacks.

This weakness maps to OWASP A02:2021 — Cryptographic Failures. The fix is to use an authenticated encryption mode like AES/GCM/NoPadding, which provides both confidentiality and integrity in a single operation — eliminating padding oracle attacks entirely.

Examples of the Issue

Dangerous: AES in ECB mode — leaks plaintext patterns (Java)

ECB (Electronic Codebook) mode encrypts each 16-byte block independently with the same key. Identical plaintext blocks produce identical ciphertext blocks, revealing patterns in the data. This is why encrypted images in ECB mode still show the outline of the original image — the famous "ECB penguin" demonstration. Using Cipher.getInstance("AES") without specifying a mode defaults to ECB on most JVMs.

// DANGEROUS: AES with no mode specified — defaults to ECB on most JVMs. public class InsecureEncryption { public static byte[] encrypt(byte[] plaintext, SecretKey key) throws Exception { // "AES" alone = AES/ECB/PKCS5Padding on most providers. // ECB encrypts each block independently — identical blocks produce // identical ciphertext, leaking structural patterns. Cipher cipher = Cipher.getInstance("AES"); // Defaults to ECB! cipher.init(Cipher.ENCRYPT_MODE, key); return cipher.doFinal(plaintext); } } // Also dangerous — explicitly requesting ECB: Cipher cipher = Cipher.getInstance("AES/ECB/PKCS5Padding"); // Never use ECB!

Dangerous: AES/CBC with PKCS5Padding but no integrity check (Java)

CBC mode with PKCS5Padding is better than ECB, but without an authentication tag (HMAC), it is vulnerable to padding oracle attacks. An attacker who can submit modified ciphertexts and observe whether the server returns a padding error vs. an application error can decrypt the entire message — one byte at a time — without knowing the key.

// DANGEROUS: AES/CBC without authentication — vulnerable to padding oracle. public class CbcEncryption { public static byte[] encrypt(byte[] plaintext, SecretKey key) throws Exception { Cipher cipher = Cipher.getInstance("AES/CBC/PKCS5Padding"); cipher.init(Cipher.ENCRYPT_MODE, key); byte[] iv = cipher.getIV(); byte[] ciphertext = cipher.doFinal(plaintext); // Returning IV + ciphertext, but NO authentication tag. // An attacker can modify the ciphertext and exploit padding errors. ByteBuffer buffer = ByteBuffer.allocate(iv.length + ciphertext.length); buffer.put(iv); buffer.put(ciphertext); return buffer.array(); } public static byte[] decrypt(byte[] data, SecretKey key) throws Exception { ByteBuffer buffer = ByteBuffer.wrap(data); byte[] iv = new byte[16]; buffer.get(iv); byte[] ciphertext = new byte[buffer.remaining()]; buffer.get(ciphertext); Cipher cipher = Cipher.getInstance("AES/CBC/PKCS5Padding"); cipher.init(Cipher.DECRYPT_MODE, key, new IvParameterSpec(iv)); return cipher.doFinal(ciphertext); // Padding error = oracle for attacker! } }

Dangerous: Node.js deprecated createCipher with weak key derivation (JS)

The deprecated crypto.createCipher() uses a single iteration of MD5 to derive the key from a password — making it trivially brute-forceable. It also uses CBC mode without authentication, combining weak key derivation with a padding-oracle-vulnerable mode.

// DANGEROUS: Deprecated API with weak key derivation + no auth tag. const crypto = require('crypto'); function encrypt(text, password) { // createCipher is DEPRECATED — derives key via single MD5 iteration. // Uses aes-256-cbc without an authentication tag. const cipher = crypto.createCipher('aes-256-cbc', password); let encrypted = cipher.update(text, 'utf8', 'hex'); encrypted += cipher.final('hex'); return encrypted; } function decrypt(encrypted, password) { const decipher = crypto.createDecipher('aes-256-cbc', password); let decrypted = decipher.update(encrypted, 'hex', 'utf8'); decrypted += decipher.final('utf8'); // Padding error = oracle! return decrypted; }

How to Resolve

Fix (Java): Use AES/GCM/NoPadding — authenticated encryption

GCM (Galois/Counter Mode) provides authenticated encryption — it encrypts the data and generates an authentication tag in a single operation. Any tampering with the ciphertext causes decryption to fail with an AEADBadTagException, completely eliminating padding oracle attacks. Always use a random 12-byte IV and never reuse an IV with the same key.

// SAFE: AES/GCM provides confidentiality + integrity (authenticated encryption). import javax.crypto.Cipher; import javax.crypto.SecretKey; import javax.crypto.spec.GCMParameterSpec; import java.security.SecureRandom; import java.nio.ByteBuffer; public class SecureEncryption { private static final int GCM_IV_LENGTH = 12; // 96 bits — recommended for GCM private static final int GCM_TAG_LENGTH = 128; // 128-bit auth tag public static byte[] encrypt(byte[] plaintext, SecretKey key) throws Exception { // Generate a random IV for each encryption — NEVER reuse with same key. byte[] iv = new byte[GCM_IV_LENGTH]; new SecureRandom().nextBytes(iv); Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding"); cipher.init(Cipher.ENCRYPT_MODE, key, new GCMParameterSpec(GCM_TAG_LENGTH, iv)); byte[] ciphertext = cipher.doFinal(plaintext); // Prepend IV to ciphertext for storage/transmission. ByteBuffer buffer = ByteBuffer.allocate(iv.length + ciphertext.length); buffer.put(iv); buffer.put(ciphertext); // Includes the GCM auth tag automatically. return buffer.array(); } public static byte[] decrypt(byte[] data, SecretKey key) throws Exception { ByteBuffer buffer = ByteBuffer.wrap(data); byte[] iv = new byte[GCM_IV_LENGTH]; buffer.get(iv); byte[] ciphertext = new byte[buffer.remaining()]; buffer.get(ciphertext); Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding"); cipher.init(Cipher.DECRYPT_MODE, key, new GCMParameterSpec(GCM_TAG_LENGTH, iv)); // If ciphertext was tampered with, this throws AEADBadTagException. return cipher.doFinal(ciphertext); } }

Fix (JS): Node.js crypto with aes-256-gcm + proper key derivation

Use crypto.createCipheriv() with aes-256-gcm instead of the deprecated createCipher(). Derive keys from passwords using scrypt or pbkdf2 with a random salt. Store the IV, salt, and auth tag alongside the ciphertext.

// SAFE: AES-256-GCM with proper key derivation and random IV. const crypto = require('crypto'); const ALGORITHM = 'aes-256-gcm'; const IV_LENGTH = 12; // 96 bits for GCM const SALT_LENGTH = 16; // 128-bit salt const TAG_LENGTH = 16; // 128-bit auth tag const KEY_LENGTH = 32; // 256-bit key /** Derive a key from a password using scrypt. */ function deriveKey(password, salt) { return crypto.scryptSync(password, salt, KEY_LENGTH); } function encrypt(text, password) { const salt = crypto.randomBytes(SALT_LENGTH); const key = deriveKey(password, salt); const iv = crypto.randomBytes(IV_LENGTH); const cipher = crypto.createCipheriv(ALGORITHM, key, iv); const encrypted = Buffer.concat([cipher.update(text, 'utf8'), cipher.final()]); const tag = cipher.getAuthTag(); // Pack: salt + iv + tag + ciphertext return Buffer.concat([salt, iv, tag, encrypted]).toString('base64'); } function decrypt(packed, password) { const data = Buffer.from(packed, 'base64'); const salt = data.subarray(0, SALT_LENGTH); const iv = data.subarray(SALT_LENGTH, SALT_LENGTH + IV_LENGTH); const tag = data.subarray(SALT_LENGTH + IV_LENGTH, SALT_LENGTH + IV_LENGTH + TAG_LENGTH); const ciphertext = data.subarray(SALT_LENGTH + IV_LENGTH + TAG_LENGTH); const key = deriveKey(password, salt); const decipher = crypto.createDecipheriv(ALGORITHM, key, iv); decipher.setAuthTag(tag); // Throws if ciphertext was tampered with. return Buffer.concat([decipher.update(ciphertext), decipher.final()]).toString('utf8'); }

Fix (Java): Generate strong AES keys with KeyGenerator

Never derive AES keys from simple strings or short passwords without a proper KDF. Use KeyGenerator for random key generation, or PBKDF2 with a high iteration count when deriving from a password. Always use 256-bit keys when possible.

// SAFE: Proper AES key generation and password-based key derivation. import javax.crypto.KeyGenerator; import javax.crypto.SecretKey; import javax.crypto.SecretKeyFactory; import javax.crypto.spec.PBEKeySpec; import javax.crypto.spec.SecretKeySpec; import java.security.SecureRandom; public class KeyUtils { /** Generate a random 256-bit AES key. Use when no password is involved. */ public static SecretKey generateRandomKey() throws Exception { KeyGenerator keyGen = KeyGenerator.getInstance("AES"); keyGen.init(256, new SecureRandom()); return keyGen.generateKey(); } /** Derive an AES key from a password using PBKDF2. */ public static SecretKey deriveKeyFromPassword(String password, byte[] salt) throws Exception { PBEKeySpec spec = new PBEKeySpec( password.toCharArray(), salt, 310_000, // OWASP recommended minimum iterations for PBKDF2-SHA256 256 // Key length in bits ); SecretKeyFactory factory = SecretKeyFactory.getInstance("PBKDF2WithHmacSHA256"); byte[] keyBytes = factory.generateSecret(spec).getEncoded(); spec.clearPassword(); // Wipe password from memory. return new SecretKeySpec(keyBytes, "AES"); } /** Generate a random salt for key derivation. */ public static byte[] generateSalt() { byte[] salt = new byte[16]; // 128-bit salt new SecureRandom().nextBytes(salt); return salt; } }
